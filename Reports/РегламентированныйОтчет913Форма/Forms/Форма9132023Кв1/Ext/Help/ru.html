Отчет «Декларация для налогоплательщиков, применяющих специальный налоговый режим розничного налога, форма 913.00 (XML)» за 2023 год.

Отчет предназначен для налогоплательщиков, применяющих специальный налоговый режим розничного налога (форма 913.00)

Правила составления:

- Правила составления налоговой отчетности по розничному налогу (Форма 913.00) за 2023 год.

Специальный налоговый режим розничного налога применяют субъекты малого бизнеса, определенные ст. 696-3 Налогового Кодекса РК, соответствующие установленным ограничениям:

  • Индивидуальные предприниматели - в справочнике "Организации" для них реквизит "Юр. / физ. лицо" должен быть установлен в значение "Физ. лицо";
  • Юридические лица - в справочнике "Организации" для них реквизит "Юр. / физ. лицо" должен быть установлен в значение "Юр. лицо".

- ИСЧИСЛЕНИЕ НАЛОГОВЫХ ОБЯЗАТЕЛЬСТВ ПО НАЛОГОПЛАТЕЛЬЩИКАМ, ПРИМЕНЯЮЩИХ СПЕЦИАЛЬНЫЙ НАЛОГОВЫЙ РЕЖИМ РОЗНИЧНОГО НАЛОГА (форма 913.01)

Декларация для налогоплательщиков, применяющих специальный налоговый режим розничного налога (форма 913.00)

Раздел. Исчисленные налоги

- Заполнение строки 913.00.001

В строке 913.00.001 выводится сумма налогооблагаемого дохода, полученного за налоговый период, определяемая как разность кредитового и дебетового оборотов по счетам разделов 6000, 6100, 6200, 6300 и 6400, при этом проводка Дт 6ххх - Кт 5710 (Итоговая прибыль итоговый убыток), формируемая при реформации баланса в момент закрытия месяца, не учитывается. 
В строку переносится итоговое значение графы H приложения 913.01.

Подстрока 913.00.001 I автоматически не заполняется.

- Заполнение строки 913.00.002

В строке указывается начисленный доход физическим лицам за соответствующий налоговый период. В строку переносится значение строки 913.01.001 приложения 913.01.

- Заполнение строки 913.00.003

В строке выводятся данные о среднесписочном количестве работников за налоговый период. Количество работников  определяется как :

                       Численность1 + Численность2 + Численность3 

913.00.003 = -------------------------------------------------------------

        3

где Численность - списочная численность работников за каждый месяц отчетного квартала. При подсчете численности учитывается количество дней, в течение которых действовал трудовой договор работника в месяце, в отношении к общему числу дней в месяце.

- Заполнение строки 913.00.004

В строке выводится итоговая сумма исчисленных налогов графы К приложения 913.01.


Раздел. Сведения о расходах, запасах

- Заполнение строк 913.00.005 и 913.00.006

Строки заполняются вручную. 

Исчисление налоговых обязательств по налогоплательщикам, применяющим специальный налоговый режим розничного налога (форма 913.01)

Приложение 913.01 автоматически заполняется данными для исчисления розничного налога.

Приложение заполняется в разрезе организаций, подразделений (если включен учет доходов по подразделениям) и ставок розничного налога.

В разделе "Общая информация о налогоплательщике" Количество мест осуществления деятельности заполняется в соответствии с количеством различных подразделений, указанных в качестве субконто "Подразделение" на счетах учета доходов, либо организаций (при незаполненном субконто, или его отсутствии).

- Заполнение строки 913.01.001

В строке отражается сумма расходов работодателя по доходам его работников за соответствующий налоговый период.

- Заполнение графы А (№)

В графе выводится номер строки по порядку в дополнительной форме.

- Заполнение графы B (код УГД)

В графе заполняется код органа государственных доходов из элемента справочника "Контрагенты", который выбран в качестве налогового комитета у подразделения (либо организации, при незаполненном субконто "Подразделение", или его отсутствии).

- Заполнение граф: C (город (аул, село)), D (улица (микрорайон)), Е (номер дома), F (номер квартиры (комнаты, иного помещения))

В графах выводятся данные регистра сведений "Контактная информация" по подразделению (либо организации, при незаполненном субконто "Подразделение", или его отсутствии). Для юридических лиц берется вид контактной информации: "Юридический адрес организации", для индивидуального предпринимателя - "Адрес по прописке физ. лица".

- Заполнение графы G (Ставка, %)

В графе заполняется ставка из регистра сведений "Сведения о ставках розничного налога", указанная для организации (СП/подразделения) в целом, либо для вида дохода.

- Заполнение графы H (Сумма облагаемого дохода)

В графе указывается сумма облагаемых доходов по ставке, указанной в графе G.

- Заполнение графы I (Удельный вес облагаемого дохода, %)

В графе отражается удельный вес облагаемого дохода в общей сумме облагаемого дохода, отраженной в итоге графы Н. 

- Заполнение графы J (Сумма дохода, с учетом уменьшения на соответствующую сумму расходов работодателя по доходам его работников)

Графа заполняется значением графы Н за вычетом суммы расходов работодателя по доходам его работников пропорционально удельному весу.

- Заполнение графы K (Сумма исчисленных налогов (КПН/ИПН))

В графе отражаются суммы исчисленных налогов (КПН/ИПН), которые определяются следующим образом: сумма дохода, с учетом уменьшения на сумму расходов работодателя по доходам работников (J) умноженная на ставку, отраженную в графе G.

Внимание! После автоматического заполнения приложения 913.01 и расчета удельного веса, итог графы I приводится к значению 100%. В связи с этим могут образоваться погрешности округления при расчете граф I и J. Эти погрешности относятся в строку с наибольшим удельным весом. После чего пересчитывается графа К по данной строке. 

Особенности ручного ввода

При изменении ячейки 913.01.001 для каждой строки рассчитываются:

  • графа I (Удельный вес)
  • графа J (Облагаемая сумма)
  • графа K (Исчисленный налог)

При измении графы G (Ставка налога) для текущей строки пересчитывается графа K (Исчисленный налог)

При изменении графы H (Доход) или I (Удельный вес) для текущей строки пересчитываются:

  • графа J (Облагаемая сумма)
  • графа K (Исчисленный налог)

Графы J и K в строке недоступны для изменения (рассчитываются по значениям других строк).


После пересчета строк, итоги по графам рассчитываются автоматически.

При ручном пересчете распределение погрешностей от округления не выполняется.
